# スライム討伐戦（疑似3Dデモ）

妖怪ウォッチバスターズ風の「自由に動いてボスに近づいて殴る」協力バトルの土台として作った、疑似3Dのボス戦デモです。
現状はサーバー接続なしの**単体動作版**（ダメージ計算もすべてフロント内で完結）です。

## 動かし方

1. このフォルダごと展開する
2. `index.html` をブラウザで開く（ダブルクリックでOK）
3. 画面下の十字ボタン（▲◀▶▼）で移動、ボスに近づくと「こうげき！」ボタンが赤く光るので押す
   - PCなら矢印キー / WASD で移動、スペースキーで攻撃も可能

ネットワーク接続も外部サーバーも不要です。CDN（Three.js本体のみ）にだけアクセスします。

## ファイル構成

```
slime_boss_battle/
├── index.html   … 画面のHTML構造のみ
├── style.css    … 見た目のスタイル
├── config.js    … 数値・調整パラメータ（ここを書き換えるだけでバランス調整できる）
├── game.js      … ゲームロジック本体
└── README.md    … このファイル
```

意図的に「設定値（config.js）」と「ロジック（game.js）」を分離しています。
HPやダメージ量、移動速度、カメラの位置などを変えたいときは `config.js` の数値をいじるだけで反映されます。

## 現状でできること

- 上下左右の自由移動（プレイヤーは常にカメラの正面手前にいるTPS風の低めカメラ）
- ボスは定位置を中心にランダムに徘徊（3.5秒ごとに目的地を変更）、ふわふわ浮遊演出つき
- 攻撃範囲を地面のリングで可視化、範囲内に入ると赤くハイライト
- 攻撃ヒット時にボスが一瞬縮む演出、HPバーの色がHP割合に応じて変化（緑〜オレンジ〜赤）
- 与ダメージ合計・攻撃回数のステータス表示
- 討伐すると「ボス討伐！」表示、攻撃ボタンが無効化
- リセットボタンで最初からやり直し

## 今後マルチプレイ化する場合の方針

`game.js` の `attackBoss()` 関数内、「ここから先がサーバー側で行うべき計算」とコメントしてある部分が
ダメージ計算のロジックです。ここを丸ごと WebSocket（STOMP）送信に差し替えれば、
サーバー側でダメージ計算・HP管理を行う構成に移行できます。

想定している移行イメージ:

1. クライアントは「攻撃した」というイベント（プレイヤーID・座標）だけをサーバーに送る
2. サーバー側でプレイヤーとボスの距離を再判定し、ダメージ計算・HP更新を行う（`@Transactional`）
3. コミット後に最新HPを `/topic/battle/{sessionId}` へブロードキャストし、全クライアントのHPバーを同期する
4. 同時に複数人が動き回れるよう、各プレイヤーの座標も一定間隔（150ms目安）でサーバーに送信・配信する

クライアント側でダメージ計算を完結させない（最終判定は必ずサーバー）ことで、チート対策にもなります。

## 既知の制約・今後の改善候補

- ボスもプレイヤーも単純な球体メッシュ。将来的にはスライムらしい3Dモデル（.glb）への差し替えを想定
- 攻撃モーションが未実装（現状は範囲内でボタンを押すだけ）。必殺技・属性などRPG要素は未着手
- 複数プレイヤーの同時参加・同期は未実装（現状は1人プレイのみ）
- カメラはプレイヤーの背後に固定。スマホでの視点回転（スワイプでカメラ操作）は未対応

## 技術スタック

- Three.js r128（CDN経由、`cdnjs.cloudflare.com` から読み込み）
- Vanilla JavaScript（フレームワーク不使用）
- 外部通信・ビルドツール一切なし。HTML/CSS/JSのみで完結
